# Online Food Delivery System - SQL Implementation
Project Overview
This project focuses on the design and implementation of a relational database for a 3-sided marketplace (Customers, Restaurants, and Delivery). The goal was to translate a conceptual ER diagram into a functional SQL database capable of handling real-world logistics data.

Key Features
Relational Schema: Designed tables for Customers, Orders, Menu Items, and Reviews with full referential integrity (Primary & Foreign Keys).

Data Integrity: Implemented CHECK constraints for ratings and UNIQUE constraints for user data.

Advanced Querying:

Used Multi-table Joins to generate comprehensive order reports.

Applied Aggregate Functions (AVG, COUNT, SUM) for restaurant performance metrics.

Implemented CASE WHEN logic for dynamic order categorization.

Used COALESCE for clean data reporting by handling null values.
